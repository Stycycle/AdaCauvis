# 48GB 服务器实验运行指南

本指南用于在 `vGPU-48GB x 1` 服务器上运行 Cauvis 对比实验。该服务器显存充足，不需要使用低显存配置。建议先跑 `4 epoch` 快速对比，确认有提升趋势后，再跑完整 `12 epoch` 正式实验。

请使用原始 `640x640` 配置：

```text
configs/cauvis/cauvis_dinov2_dinohead_bs1x4_sdgod.py
```

不要使用：

```text
configs/cauvis/cauvis_dinov2_dinohead_bs1x1_sdgod_384_lowmem.py
```

也不要再加低显存参数：

```text
model.num_queries=100
model.backbone.cauvis_config.token_length=256
```

## 0. 进入项目目录

作用：确保后续命令都在服务器项目根目录执行。

```bash
cd /root/autodl-tmp/Cauvis
```

## 1. 快速训练 Baseline：固定 0.20，4 epoch

作用：训练对照组。所有 Transformer 层都使用固定 `0.20` 低频保留比例。默认 `4 epoch`、`batch_size=4`，用于快速看趋势，不作为最终正式结果。

```bash
bash scripts/train_fixed_020_ep4_bs4.sh
```

产物：

```text
work_dir/train_fixed_020_640_ep4_bs4/epoch_4.pth
```

## 2. 快速训练改进版：Layer-Adaptive，4 epoch

作用：训练改进组。默认使用浅层 `0.15` 到深层 `0.42` 的分层频域比例。它必须和 Baseline 使用同样的 epoch、数据、分辨率和 batch 设置。

```bash
bash scripts/train_layer_adaptive_ep4_bs4.sh
```

产物：

```text
work_dir/train_layer_adaptive_640_ep4_bs4/epoch_4.pth
```

## 3. 测试 Baseline，4 epoch checkpoint

作用：用 Baseline checkpoint 在完整 SDGOD 测试集上评估。

```bash
bash scripts/test_fixed_020_ep4_bs4.sh
```

产物：

```text
work_dir/test_fixed_020_640_ep4_bs4/<timestamp>/<timestamp>.json
```

## 4. 测试改进版，4 epoch checkpoint

作用：用 Layer-Adaptive checkpoint 在同一套完整 SDGOD 测试集上评估。

```bash
bash scripts/test_layer_adaptive_ep4_bs4.sh
```

产物：

```text
work_dir/test_layer_adaptive_640_ep4_bs4/<timestamp>/<timestamp>.json
```

## 5. 查找测试结果

作用：找到两个测试任务生成的 JSON 指标文件。

```bash
bash scripts/find_ep4_bs4_results.sh
```

## 6. 比较指标

重点比较：

```text
pascal_voc/daytime_clear_mAP
pascal_voc/daytime_foggy_mAP
pascal_voc/dusk_rainy_mAP
pascal_voc/night_rainy_mAP
pascal_voc/Night-Sunny_mAP
pascal_voc/mean_mAP
```

整理成表：

```text
Method              Clear  Foggy  DuskRain  NightRain  NightSunny  Mean
Fixed 0.20
Layer-Adaptive
Delta
```

判断标准：

- `mean_mAP` 提升：整体有效。
- `dusk_rainy`、`night_rainy` 提升：频域分层对恶劣天气有效。
- `daytime_clear` 明显下降：滤波可能损伤源域，需要降低强度。
- 指标几乎不变：固定 schedule 作用有限，后续考虑 learnable ratio 或动态光照。

## 7. 如果 4 epoch 有提升，再跑 12 epoch

作用：获得更可靠的正式对比结果。只需要把前面训练命令里的三项改掉：

```text
train_cfg.max_epochs=12
train_cfg.val_interval=12
work-dir 去掉 _ep4_bs4，例如 work_dir/train_fixed_020_640
```

对应 checkpoint 会变成：

```text
work_dir/train_fixed_020_640/epoch_12.pth
work_dir/train_layer_adaptive_640/epoch_12.pth
```

测试命令也把 checkpoint 路径从 `epoch_4.pth` 换成 `epoch_12.pth`，并使用新的测试目录，例如：

```text
work_dir/test_fixed_020_640
work_dir/test_layer_adaptive_640
```

## 8. 注意事项

- 服务器只有 1 张 GPU，因此使用 `python tools/train.py`，不是 `tools/dist_train.sh ... 8`。
- `XFORMERS_DISABLED=1` 是为了避免 xFormers/CUDA 版本不匹配；48GB 显存足够时禁用 xFormers 更稳。
- Baseline 和 Layer-Adaptive 必须使用相同配置，只允许频域比例不同。
- `default_hooks.checkpoint.interval=1` 会每个 epoch 保存一次 checkpoint，避免中途断开后无法续跑。
- 如果服务器 xFormers 与 CUDA 12.8 完全匹配，可以尝试去掉 `XFORMERS_DISABLED=1`，但第一次建议保留。
