# 低显存实验运行指南

本指南用于在 6GB 左右显存环境下验证 Layer-Adaptive Fourier Filtering 是否比固定 0.20 频域比例更好。该实验使用低显存设置：`384x384` 输入、`batch_size=1`、`num_queries=100`、`token_length=256`。它用于判断方向是否有信号，不作为最终论文主结果。

## 1. 训练 Baseline：固定 0.20 频域比例

作用：训练对照组。该设置让所有 Transformer 层都使用固定 `0.20` 低频保留比例。

```bash
XFORMERS_DISABLED=1 PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True CUDA_VISIBLE_DEVICES=0 \
python tools/train.py \
  configs/cauvis/cauvis_dinov2_dinohead_bs1x1_sdgod_384_lowmem.py \
  --amp \
  --work-dir work_dir/train_fixed_020_384_tiny \
  --cfg-options \
  model.num_queries=100 \
  model.backbone.cauvis_config.token_length=256 \
  model.backbone.cauvis_config.min_low_freq_ratio=0.2 \
  model.backbone.cauvis_config.max_low_freq_ratio=0.2
```

产物：`work_dir/train_fixed_020_384_tiny/epoch_12.pth`

## 2. 训练改进版：Layer-Adaptive 频域比例

作用：训练改进组。该设置使用默认分层频域比例，浅层从 `0.15` 开始，深层逐步增加到 `0.42`。

```bash
XFORMERS_DISABLED=1 PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True CUDA_VISIBLE_DEVICES=0 \
python tools/train.py \
  configs/cauvis/cauvis_dinov2_dinohead_bs1x1_sdgod_384_lowmem.py \
  --amp \
  --work-dir work_dir/train_layer_adaptive_384_tiny \
  --cfg-options \
  model.num_queries=100 \
  model.backbone.cauvis_config.token_length=256
```

产物：`work_dir/train_layer_adaptive_384_tiny/epoch_12.pth`

## 3. 测试 Baseline

作用：用 Baseline checkpoint 在 5 个 SDGOD 测试域上评估，得到对照组 mAP。

```bash
XFORMERS_DISABLED=1 CUDA_VISIBLE_DEVICES=0 \
python tools/test.py \
  configs/cauvis/cauvis_dinov2_dinohead_bs1x1_sdgod_384_lowmem.py \
  work_dir/train_fixed_020_384_tiny/epoch_12.pth \
  --work-dir work_dir/test_fixed_020_384_tiny \
  --cfg-options \
  model.num_queries=100 \
  model.backbone.cauvis_config.token_length=256 \
  model.backbone.cauvis_config.min_low_freq_ratio=0.2 \
  model.backbone.cauvis_config.max_low_freq_ratio=0.2
```

产物：`work_dir/test_fixed_020_384_tiny/<timestamp>/<timestamp>.json`

## 4. 测试改进版

作用：用 Layer-Adaptive checkpoint 在同一套测试域上评估，得到改进组 mAP。

```bash
XFORMERS_DISABLED=1 CUDA_VISIBLE_DEVICES=0 \
python tools/test.py \
  configs/cauvis/cauvis_dinov2_dinohead_bs1x1_sdgod_384_lowmem.py \
  work_dir/train_layer_adaptive_384_tiny/epoch_12.pth \
  --work-dir work_dir/test_layer_adaptive_384_tiny \
  --cfg-options \
  model.num_queries=100 \
  model.backbone.cauvis_config.token_length=256
```

产物：`work_dir/test_layer_adaptive_384_tiny/<timestamp>/<timestamp>.json`

## 5. 查找测试结果文件

作用：找到两个测试任务生成的 JSON 指标文件。

```bash
find work_dir/test_fixed_020_384_tiny work_dir/test_layer_adaptive_384_tiny \
  -name "*.json" -print
```

## 6. 比较结果

重点比较以下指标：

```text
pascal_voc/daytime_clear_mAP
pascal_voc/daytime_foggy_mAP
pascal_voc/dusk_rainy_mAP
pascal_voc/night_rainy_mAP
pascal_voc/Night-Sunny_mAP
pascal_voc/mean_mAP
```

建议整理成表：

```text
Method              Clear  Foggy  DuskRain  NightRain  NightSunny  Mean
Fixed 0.20
Layer-Adaptive
Delta
```

判断标准：
- `mean_mAP` 提升：说明整体有效。
- `dusk_rainy`、`night_rainy` 提升：说明频域分层对恶劣天气有帮助。
- `daytime_clear` 明显下降：说明滤波可能损伤源域，需要调弱。
- 所有指标几乎不变：说明固定 schedule 作用有限，可以考虑 learnable ratio 或动态光照方向。

## 7. 注意事项

- Baseline 和 Layer-Adaptive 必须使用相同低显存设置，只允许频域 ratio 不同。
- 如果训练中断，先看对应 `work_dir/.../<timestamp>.log` 的最后 100 行。
- 如果仍然 OOM，可以临时继续降低 `model.num_queries`，但两组实验必须保持一致。
- 当前低显存实验只能判断方向，不代表最终论文性能；最终结果仍应在原始 `640x640` 配置和足够显存上复现。
