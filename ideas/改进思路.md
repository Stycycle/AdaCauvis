# Cauvis 改进思路

## 项目背景

Cauvis 是一个单域泛化目标检测（Single-Domain Generalized Object Detection）方法：
- **训练**：只在晴天白天（daytime_clear）数据上训练
- **测试**：直接泛化到雾天、雨天、夜间等恶劣天气
- **核心**：在 DINOv2 ViT 每层后插入轻量 Cauvis 模块，冻结预训练权重，只训练 Cauvis（~1M 参数）

---

## 原始 Cauvis 机制

```
输入特征 feats (B, L+1, C)
    ↓
去掉 cls_token
    ↓
Cross-Attention with Prompt (可学习参数)
    ↓
双分支处理：
  ├── Causal Branch (MLP) × α      → 学习域不变特征
  └── Auxiliary Branch (FFT+低通) × β  → 去除高频噪声
    ↓
残差融合：feats × δ + main + aux
    ↓
拼接 cls_token
```

**关键参数**：
- `prompt`: 可学习参数 (100, C)
- `α, β, δ`: 可学习缩放系数（初始 0.001）
- **频域滤波**：固定保留中心 20% 低频，砍掉 80% 高频

---

## 改进方向

### 当前落地版本：Layer-Adaptive 2D Fourier Filtering

当前优先维护的改进是 **改进二：多层频域滤波**。原因是它最贴合现有 Cauvis 结构：`CauvisDINOv2.forward_features()` 已经把 Transformer 层号 `idx` 传给 `Cauvis.forward()`，原始代码却没有使用这个信息。因此先利用层号控制频域滤波强度，改动集中、风险较低，也最适合做清晰消融。

**已实现文件**：`mmdet/models/layers/cauvis.py`

**实现要点**：
- 原始固定 `20%` 频域保留比例改为分层 schedule。
- 默认从浅层 `0.15` 线性增加到深层 `0.42`。
- 对 square patch tokens 使用 `2D FFT`，更符合 ViT patch grid 的空间结构。
- 使用 `fftshift / ifftshift` 后再做中心低频 mask，修正原先直接在 `fft` 输出中心取 mask 的低频位置问题。
- 非 square token 数自动 fallback 到修正后的 `1D FFT`。

**默认配置**：

```python
cauvis_config=dict(
    type="Cauvis",
    min_low_freq_ratio=0.15,
    max_low_freq_ratio=0.42,
)
```

现有 config 未显式写这两个参数时，会使用上述默认值。

**验证记录**：
- `python -m py_compile mmdet/models/layers/cauvis.py`
- 使用最小 mock registry 跑过 `Cauvis` 前向，输入输出 shape 正常。
- 本地缺少 `mmcv`，完整 `mmdet` 配置级验证需要在训练环境中执行。

**下一步实验**：
1. 跑原始 Cauvis baseline，保留同一随机种子和训练配置。
2. 跑当前 Layer-Adaptive 版本，对比 5 个 SDGOD 域的 mAP。
3. 如果整体提升稳定，再做 `min/max ratio` 消融：`0.10→0.35`、`0.15→0.42`、`0.20→0.50`。
4. 如果提升主要集中在 rain/night 域，保留该方向并继续做 learnable ratio。

### 改进一：动态光照感知 Prompt（Illumination-Aware Dynamic Prompt）

**问题**：原始 Cauvis 的 α, β, δ 是固定标量，对所有图像一视同仁。但不同天气的退化程度不同（晴天 vs 浓雾夜间），需要不同程度的适应。

**AllWeatherNet 启发**：AllWeatherNet 用灰度图做空间注意力，暗处多增强、亮处少增强。我们将这种"光照自适应"思想迁移到特征级。

**改进**：根据输入图像的全局光照统计，动态调整三个分支的权重。

```python
class IlluminationEncoder(nn.Module):
    def __init__(self, embed_dims, hidden_dim=64):
        super().__init__()
        self.mlp = nn.Sequential(
            nn.Linear(embed_dims, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 3)  # [α_scale, β_scale, δ_scale]
        )
    
    def forward(self, cls_token):
        # cls_token: (B, C)
        return self.mlp(cls_token)  # (B, 3)


class DynamicIlluminationCauvis(nn.Module):
    def __init__(self, embed_dims, ...):
        ...
        # 新增：光照编码器
        self.illumination_encoder = IlluminationEncoder(embed_dims)
    
    def forward(self, feats, layer, ...):
        cls_token, patch_feats = torch.tensor_split(feats, [1], dim=1)
        
        # 动态调制
        illum = self.illumination_encoder(cls_token.squeeze(1))  # (B, 3)
        
        # 原始参数 + 动态调整（tanh 限制范围，防止过大）
        alpha = self.alpha * (1 + torch.tanh(illum[:, 0:1]) * 0.5)
        beta = self.beta * (1 + torch.tanh(illum[:, 1:2]) * 0.5)
        delta = self.delta_scale * (1 + torch.tanh(illum[:, 2:3]) * 0.5)
        
        # 标准流程
        res = self.cross_attention(patch_feats, self.prompt)
        main = self.prompt_branch(res) * alpha
        aux = self.aux_branch(res) * beta
        patch_feats = patch_feats * delta + main + aux
        
        return torch.cat([cls_token, patch_feats], dim=1)
```

**为什么有效**：
- 夜间图像（暗）：光照编码器输出"暗"信号 → 增强 β（去噪）、增强 α（语义）
- 雾天图像（低对比度）：调整 δ 保留更多原始特征
- 晴天图像（正常）：三个系数接近 1，不破坏预训练知识

**参数量**：仅增加 ~6K（两个 Linear）

### 改进二：多层频域滤波（Layer-Adaptive Fourier Filtering）

**问题**：原始 Cauvis 在所有 24 层 Transformer 用同一个固定比例（20%）做频域滤波。但不同层处理的信息不同：
- 浅层：边缘、纹理、颜色（高频信息多）
- 中层：部件、形状
- 深层：语义、场景（低频信息多）

固定 20% 一刀切的问题：
- 浅层：20% 太弱 → 雨滴/雪花噪声去不干净
- 深层：20% 太强 → 把语义信息也滤掉了

**改进**：不同深度用不同的频域滤波策略。

```python
class LayerAdaptiveFourier(nn.Module):
    def __init__(self, num_layers=24):
        super().__init__()
        # 每层独立学习最优滤波比例
        self.ratios = nn.ParameterList([
            nn.Parameter(torch.tensor(0.15)),  # 浅层：强去噪
            nn.Parameter(torch.tensor(0.15)),
            nn.Parameter(torch.tensor(0.18)),
            nn.Parameter(torch.tensor(0.18)),
            nn.Parameter(torch.tensor(0.20)),
            nn.Parameter(torch.tensor(0.20)),
            nn.Parameter(torch.tensor(0.22)),
            nn.Parameter(torch.tensor(0.22)),
            nn.Parameter(torch.tensor(0.25)),  # 中层：平衡
            nn.Parameter(torch.tensor(0.25)),
            nn.Parameter(torch.tensor(0.28)),
            nn.Parameter(torch.tensor(0.28)),
            nn.Parameter(torch.tensor(0.30)),
            nn.Parameter(torch.tensor(0.30)),
            nn.Parameter(torch.tensor(0.32)),
            nn.Parameter(torch.tensor(0.32)),
            nn.Parameter(torch.tensor(0.35)),  # 深层：弱滤波
            nn.Parameter(torch.tensor(0.35)),
            nn.Parameter(torch.tensor(0.38)),
            nn.Parameter(torch.tensor(0.38)),
            nn.Parameter(torch.tensor(0.40)),
            nn.Parameter(torch.tensor(0.40)),
            nn.Parameter(torch.tensor(0.42)),
            nn.Parameter(torch.tensor(0.42)),
        ])
    
    def forward(self, x, layer_idx):
        ratio = torch.sigmoid(self.ratios[layer_idx])  # 保证 0~1
        return self.fourier_filter(x, ratio)
```

**物理直觉**：

| 层深度 | 特征内容 | 天气影响 | 滤波策略 |
|--------|---------|---------|---------|
| 浅层（1-8）| 边缘、纹理 | 雨滴/雪花覆盖细节 | 强去噪，保留 15-22% 低频 |
| 中层（9-16）| 部件、形状 | 雾模糊轮廓 | 中等，保留 25-32% |
| 深层（17-24）| 语义、场景 | 整体对比度下降 | 弱滤波，保留 35-42% |

**类比**：给不同年级学生布置不同难度的作业——一年级做基础题，六年级做难题。

### 改进三：信息瓶颈频域滤波（Information Bottleneck Fourier Filtering）

**问题**：原始 Cauvis 的频域滤波是"硬截断"——固定保留中心 20%，其余全部砍掉。这种非黑即白的方式可能：
- 把有用的高频信息也砍掉了
- 对所有图像、所有频率段一视同仁

**改进**：把"固定砍 80%"变成"学习每段频率保留多少"，连续可微、自适应。

```python
class InfoBottleneckFourier(nn.Module):
    def __init__(self, embed_dims, num_bands=8):
        super().__init__()
        self.num_bands = num_bands
        
        # 从全局特征生成频段权重分布
        self.encoder = nn.Sequential(
            nn.Linear(embed_dims, 64),
            nn.ReLU(),
            nn.Linear(64, num_bands * 2)  # μ 和 logσ，用于重参数化
        )
    
    def forward(self, x):
        B, L, C = x.shape
        
        # 全局特征统计
        feat_stats = x.mean(dim=1)  # (B, C)
        
        # 生成每段频率的保留分布参数
        params = self.encoder(feat_stats)  # (B, 16)
        mu = params[:, :self.num_bands]           # (B, 8)
        log_sigma = params[:, self.num_bands:]    # (B, 8)
        
        # 重参数化采样（VAE 风格）
        eps = torch.randn_like(mu)
        weights = torch.sigmoid(mu + torch.exp(log_sigma) * eps)  # (B, 8), 0~1
        
        # FFT + 分频段加权
        fft = torch.fft.fft(x, dim=1)
        bands = torch.chunk(fft, self.num_bands, dim=1)
        
        weighted = []
        for i, band in enumerate(bands):
            w = weights[:, i:i+1, None]  # (B, 1, 1)
            weighted.append(band * w)
        
        filtered = torch.cat(weighted, dim=1)
        return torch.fft.ifft(filtered, dim=1).real
```

**关键**：
- 把频率分成 8 段，每段学一个保留比例
- 用 VAE 重参数化建模不确定性
- 每个样本、每段频率都有自适应权重
- 连续软掩码，非硬截断

**一句话**：固定 20% → 学习每段频率保留多少，每个样本不同。

## 三个改进的协同

```
输入图像
    ↓
[光照编码器] ──→ 全局退化程度 ──→ 调整 α, β, δ（改进一）
    ↓
[多层频域滤波] ──→ 每层不同策略 ──→ 浅层强去噪、深层保语义（改进二）
    ↓
[信息瓶颈频域] ──→ 每段频率自适应 ──→ 软掩码、连续权重（改进三）
    ↓
输出：天气鲁棒特征
```

| 改进 | 作用维度 | 解决的问题 |
|------|---------|----------|
| 动态光照 | 全局 / 图像级 | 不同图像需要不同适应强度 |
| 多层频域 | 层间 / 深度级 | 不同层需要不同频率处理 |
| 信息瓶颈 | 局部 / 频率级 | 每段频率需要不同保留比例 |

**1+1+1 > 3**：全局自适应 + 层间精细 + 频率连续 = 更鲁棒的特征

## 放弃的方向

### ❌ 天气感知混合专家（Weather MoE）

**想法**：4 个专家（雾/雨/雪/夜间）+ 路由器

**放弃原因**：
1. **覆盖不全**：天气类型无限（沙尘、冰雹、极光...），4 个专家远远不够
2. **过拟合风险**：专家数量 = 天气类型数，遇到新天气就懵
3. **违背单域泛化初衷**：我们就是要"没见过也能处理"，不是"记住每种天气"
4. **参数膨胀**：4 倍参数，失去 PEFT 优势

**结论**：通用处理 > 专家分类

## 实验计划

### 阶段一：基线复现
1. 复现原始 Cauvis 结果（验证环境正确）
2. 记录 5 个域的 mAP：daytime_clear, daytime_foggy, dusk_rainy, night_rainy, Night-Sunny

### 阶段二：逐个验证
3. **+ 多层频域**（改进二，已实现第一版）→ 验证分层策略有效
4. **+ learnable ratio**（改进二增强版）→ 验证固定 schedule 是否还能继续提升
5. **+ 动态光照**（改进一）→ 验证光照感知有效
6. **+ 信息瓶颈**（改进三）→ 验证自适应权重有效

### 阶段三：组合
7. **改进二 + 改进一** → 验证频域分层和光照调制的协同效果
8. **改进一 + 改进二 + 改进三** → 最终模型

### 阶段四：对比
9. 与 SOTA 对比（Reins、VPT、Adapter 等）
10. 消融实验：验证每个组件的必要性

## 代码实现优先级

1. **先做改进二**（多层频域）→ 已实现第一版，优先跑实验验证。
2. **再做改进二增强版**（learnable ratio）→ 在固定 schedule 有收益后再引入可学习比例。
3. **再做改进一**（动态光照）→ 作为第二个稳定创新点，但需要谨慎设计光照统计来源。
4. **最后做改进三**（信息瓶颈）→ 最复杂，训练稳定性风险最高，适合作为后续增强版。
